Introducción a R, parte 2¶

Bioinformtica para Antropologa

En esta ocassin vamos a trabajar con tablas de datos aplicando lo que aprendimos en el tutorial anterior.

Cuando trabajamos en R, todo lo que hacemos ocurre en un directorio (o carpeta) del sistema de archivos de la computadora al cual se le llama "directorio de trabajo". Para cargar una tabla de datos en R este archivo debe estar en el directorio de trabajo o bien darle la ubicacin con el camino completo al archivo.

Cargando archivos¶

Le podemos preguntar a R en qu directorio de trabajo nos encontramos:

In [1]:
getwd()
'/home/miguel/Documents/clases/bioinformatica-para-antropologia'

As pues, es posible cargar cualquier archivo que se encuentre en este directorio. Noten que esta ubicacin es de un sistema unix (linux y MacOS), si fuera un sistema windows, la salida se vera as:

C:/Users/Miguel/Documents/clases/bioinformatica-para-antropologia

Como se que mi tabla de datos cuestionario.cvs se encuentra en este directorio, la puedo cargar directamente usando la funcin read.table():

In [2]:
cuest <- read.table('cuestionario.csv', sep=',')

Si el archivo que me interesa se encuentra en otro directorio, lo puedo cambiar con la funcin setwd():

In [3]:
setwd('/home/miguel/Documents/clases/')
In [4]:
getwd()
'/home/miguel/Documents/clases'

Aunque se que mi tabla est en otro directoio la voy a tratar de cargar para ver que pasa:

In [5]:
cuest <- read.table('cuestionario.csv', sep=',')

Warning message in file(file, "rt"):
cannot open file 'cuestionario.csv': No such file or directory
Error in file(file, "rt"): cannot open the connection
Traceback:

1. read.table("cuestionario.csv", sep = ",")
2. file(file, "rt")

Esto es un error. Primero R nos dice a grandes razgos que es lo que sucedi y luego enumera la serie de pasos que llevaron al error. El enunciado que nos da la clave en este caso es cannot open file 'cuestionario.csv': No such file or directory ya que nos explica que el problema es que el archivo que le pedimos leer no se encuentra en el directorio de trabajo.

A veces tenemos que cargar archivos de diferentes directorios y resultara engorroso estar cambiando de directorio para cargar cada uno de ellos. Para evitar hacer eso podemos darle a R la ubicacin (o path) completa a cada archivo sin importar cual sea el directorio de trabajo:

In [6]:
cuest <- read.table('/home/miguel/Documents/clases/bioinformatica-para-antropologia/cuestionario.csv', sep=',')

Regresamos al directorio original:

In [7]:
setwd('/home/miguel/Documents/clases/bioinformatica-para-antropologia')
In [8]:
getwd()
'/home/miguel/Documents/clases/bioinformatica-para-antropologia'

Hemos estado usando la funcin read.table() para cargar la tabla de archivos pero no hemos visto todos sus argumentos clave. Por ahora solo le hemos dado el nombre del archivo a cargar y el separador, que es el caracter en el archivo de texto plano que delimita las columnas. En este caso el separador es una coma.

Veamos qu clase tiene el objeto cuest:

In [9]:
class(cuest)
'data.frame'

Este tipo de objeto es tal vez el ms importante dentro de R y es una matriz en la cual cada fila es una observacin o muestra y cada columna es una variable. Muchas fuciones y operadores de R funcionan con objetos de tipo data frame.

Aqu R nos dice que tenemos una tabla con 35 observaciones y 5 variables, adems dice que las 5 variables son de tipo Factor, que se refiere a variables categricas. Asimismo, R nombr cada variable arbitrariamente con una "V" y un nmero. Sin embargo, en la tabla nosotros queramos solo 4 variables de las cuales 3 eran nmricas y una era categrica. Qu sucedi?

Como se mencion en el tutorial anterior, R es un lenguaje de programacin y por consiguiente debemos ser completamente explcitos en lo que le estamos pidiendo. R no tuvo forma de saber que la primera fila del archivo de texto corresponda a los nombres de las columnas, as que al ver letras, decidi que las columnas correspondan a factores y no a caracteres numricos.

Vamos a volver a pedirle a R que lea la tabla pero en esta ocasin vamos a decirle que la primera fila correponde a los nombres de las columnas. Esto lo vamos a lograr agregando el argumento header que funciona con datos lgicos (booleanos) que son TRUE (o simplemente T) o FALSE (o F):

In [14]:
cuest <- read.table('cuestionario.csv', sep=',', header=T)

Como pueden ver, R nombr las columnas o variables del data frame segn el encabezado de cada columna y ya reconoci la estatura y la talla como datos numricos. Noten que en la edad dice int que se refiere a integer que es la forma de denominarle a los nmeros enteros. Generalmente las computadoras ven a los enteros y a los decimales como cosas distintas porque trabajar con enteros consume menos memoria que trabajar con decimales.

Ahora que tenemos una tabla limpia vamos a revisar nuestros datos de manera grfica. Primero vamos a declarar el tamao de nuestras grficas con la funcin options():

In [43]:

Esta grfica es poco informativa ya no tiene ttulo y el eje y no tiene escala (como las grficas que se usaron en los debates de candidatos presidenciables en 2018). Vamos a agregarlas con el argumento main y xlab:

In [44]:
barplot(table(cuest$X.Le.gusta.la.pizza.con.pia.), main="Le gusta la pizza con pia?", ylab="Nmero de personas")

In [45]:

Podemos replicar esta grfica cambiando un poco la sintxis de los argumentos de la funcin plot. Con tal fin, vamos a hacer explcita la relacin que creemos que existe entre la estatura y la talla del calzado usando el operador "~" que se lee como "se distribuye en funcin de":

In [46]:
plot(cuest$Talla.de.calzado.en.centmetros.~cuest$Estatura.en.centmetros.)

Ahora bien, tambin podemos simplificar un poco la llamada a los datos cuando estos se encuentran en un mismo data frame usando el argumento data:
